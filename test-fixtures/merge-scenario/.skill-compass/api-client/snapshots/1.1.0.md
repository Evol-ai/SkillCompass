# API Client Generator

## Prerequisites

- OpenAPI 3.0+ or Swagger 2.0 spec file (JSON or YAML)
- Target language runtime installed (Node.js / Python / Go)

## How to use

### Step 1: Load and validate spec

Read the spec file. Validate structure and resolve all `$ref` references.

If validation fails, report specific errors with JSON path locations.

### Step 2: Choose target language

Ask: "Generate client in TypeScript, Python, or Go?"

### Step 3: Generate client

For each endpoint in `paths`, generate a typed function:

- Method name from `operationId`
- Fully typed request/response from schema components
- Auth injection from `securitySchemes`

### Step 4: Add retry and rate limiting

Generate middleware for:
- Exponential backoff retry on 429/5xx responses
- Rate limit tracking from response headers

### Step 5: Write output

Write to user-specified directory with proper project structure.

## Output Format

```
Generated: <count> endpoint functions
Language: <target>
Auth: <scheme_type>
Retry: enabled (429/5xx)
Output: <directory>
```

## Edge Cases

- Missing operationId: derive from method + path
- Circular $ref: detect and generate partial types with TODO markers
- Discriminated unions: map oneOf/anyOf to language union types
- Large specs (>100 endpoints): filter by tag
