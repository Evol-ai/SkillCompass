# 中文翻译助手

## 前提条件

- 输入文件必须是 Markdown 格式
- 建议准备术语表（可选）

## 使用方法

### 第一步：加载文件

使用 **Read** 工具读取目标文件。检测文件语言：
- 英文为主 → 翻译为中文
- 已经是中文 → 报告"文件已是中文，无需翻译"

### 第二步：保护不翻译区域

标记以下区域，翻译时跳过：
- 代码块（``` 包裹的内容）
- 行内代码（` 包裹的内容）
- URL 和链接地址
- 变量名、函数名、类名
- frontmatter 中的 name 字段

### 第三步：加载术语表

如果项目目录存在 `.translation-glossary.json`，加载术语映射：

```json
{
  "component": "组件",
  "middleware": "中间件",
  "hook": "钩子",
  "dependency injection": "依赖注入"
}
```

术语表中的映射优先于 AI 翻译。

### 第四步：翻译

逐段翻译，保持：
- Markdown 标题层级不变
- 列表格式不变
- 表格结构不变
- 代码注释翻译（如果用户要求）

翻译风格：
- 技术文档风格，简洁准确
- 长句拆分，符合中文阅读习惯
- 被动语态转主动语态

### 第五步：输出

询问用户："覆盖原文件还是生成新文件？（默认：生成 `<filename>.zh.md`）"

使用 **Write** 工具保存。

## 输出格式

```
翻译完成：<文件名>
- 段落数：<数量>
- 保护区域：<代码块数> 个代码块，<行内代码数> 个行内代码
- 术语表匹配：<数量> 个术语
- 输出：<文件路径>
```

## 边界情况

- **中英混合文档**：只翻译英文段落，中文段落保持不变
- **无术语表**：正常翻译，使用通用翻译
- **超长文档（>500段）**：分批处理，每批 50 段
- **翻译不确定**：对不确定的翻译标注 `[?]`，让用户审查
